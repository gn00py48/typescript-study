### typescript 개론
1. typeScript = 오픈소스
2. typescript = js의 확장판
2-1. 왜 ts가 필요할까?
js는 웹 브라우저에서의 기본적인 상호작용을 위해서 탄생한 언어로, 언어의 접근성 및 편리성에 집중해서 개발되어 버그 방지 및 안정성은 후순위로 개발됨.
해당 단점을 커버허기 위해 점진적 타입 시스템을 탑제한 탄생한 언어가 ts 입니다.
3. js와 ts의 가장 큰 차이점 : 타입 시스템
3-1. 타입 시스템이란?
"언어에서 사용할 수 있는 아주 여러가지 값들을 어떤 기준으로 묶어서 타입으로 정할지 결정하고 또 코드의 타입을 언제 검사할지 그리고 어떻게 검사할지 등의 우리가 프로그래밍 언어를 사용할때 타입과 관련해서 지켜야 하는 규칙들을 모아둔 체계"
3-2. 타입 시스템의 종류
정적 타입 시스템 - java, c / 동적 타입 시스템 - python, js/ 점진적 타입 시스템 - ts
4. ts의 구동 원리
4-1. 일반적인 프로그래밍 언어의 구동 방식은 컴파일 순서에 따라
인간 : 프로그래밍 언어 -> 컴파일러 -> 기계어 : 컴퓨터
구조로 진행됩니다.
이 때, 컴파일러에서는 ast라는 구조로 프로그래밍 언어를 분해하고, 이후 기계어로 변환합니다.
js 또한 해당 구조를 따라 js -> ast -> 기계어 순서를 거친다.
4-2. ts의 구동 과정
ts의 경우, 일반적인 프로그래밍 언어와 달리 js로의 변환 과정을 거치게 됩니다.
ts의 경우 js의 확장판으로, js의 안정성을 위한 기능들이 합쳐진 언어이기 때문에 ts의 안정성 부분을 해체하면 js가 되는 구조를 가지고,
ts -> ast 변환 이후 타입 시스템을 포함한 컴파일 과정에서 오류가 없다면 js로 변환하여 이후 기계어 변환 과정을 거치게 됩니다.
즉, ts의 경우 ts->ast->js(or 컴파일 실패 exit)->기계어의 과정을 거치게 되는 것입니다.
